1. HTML:

  1. Tags Básicas

As tags básicas de um documento HTML são:

  • < !DOCTYPE html>: informa o navegador que é um documento HTML5;
  • < html>: define o início e o fim de um documento HTML;
  • < head>: define a seção de cabeçalho onde colocamos o título, link da fonte, entre outros;
  • < meta>: permite inserir metadados no documento;
  • < title>: define o título da página que aparece na guia do seu navegador;
  • < body>: define a seção do corpo do documento , onde todo o conteúdo visível da página é colocado.

  1. Tags de Texto

Essa é a tag < p>. Ela define um parágrafo;

O < span> é usado para colocar estilização; O < b> deixa o texto em negrito; O < i> deixa o texto em itálico; O < strong> indica que é algo importante e deixa o texto em negrito.
  1. Tags Semânticas

São tags com um significado claro em relação ao conteúdo que elas envolvem:

  • < header> : define o cabeçalho da página, geralmente contém o logo, o título, etc;
  • < nav> : define uma seção de navegação da página;
  • < main> : define a seção principal do conteúdo da página;
  • < section> : define uma seção de conteúdos da página;
  • < main> : define uma seção de conteúdo independente e completo, um post, por exemplo;
  • < aside> : define uma seção com conteúdo complementar, um menu lateral, por exemplo;
  • < footer> : define o rodapé da página, com informações de contato, copyright, etc.
  1. Tags de Formulário

Muitas vezes são usados para capturar dados dos usuários através da web;

Pode conter campos de texto, caixa de seleção, botões, entre outros elementos, tais como:

  • < form>: define o início de um formulário;
  • < fieldset>: agrupa elementos de um formulário;
  • < legend>: nomeia e descreve o conteúdo agrupado;
  • < label>: nomeia e descreve um elemento de entrada do formulário;
  • < input>: define um campo que vai receber dados do usuário.

Exemplo de Formulário:

O elemento < input> recebe um atributo type que define a maneira com que esse campo se comporta. O tipo padrão é um campo de texto, mas temos uma variedade de outros valores:

  • password: cria um campo de entrada de senha onde o que o usuário digita é ocultado;
  • email: cria um campo de entrada de e-mail que verifica se o valor preenchido é um e-mail válido;
  • date: cria um campo de entrada que só aceita datas;
  • number: cria um campo de entrada que só aceita números;
  • checkbox: cria uma caixa de marcação, onde é possível marcar/desmarcar várias opções;
  • button: cria um botão;
  • submit: é responsável por enviar os dados do formulário.

Podemos enviar os dados de um formulário de duas maneiras:

  • Passando um type="submit" para um input

    • ou

  • Adicionando um botão, pela tag < button> que por si só já é identificado como um submit.

  • < ol> : é usado para listas ordenadas (ou numeradas), pois necessitam uma ordem;

  • < li> : é usado para cada item dentro de uma lista.

2. CSS:

  1. Propriedades mais Comuns

As propriedades mais comuns são:

  • color: define a cor do texto. Seus valores possíveis são:
    • nomes de cores: color = red;
    • códigos hexadecimais (HEX): color: #FF0000;
    • valores do sistema RGB: color: rgb(255, 0, 0);
  • background-color: cor de fundo. Seus valores possíveis são os mesmo do color;
  • border: configura a borda dos elementos. Possui subpropriedades:
    • border-width: 2px;
    • border-style: dashed;
    • border-color: blue;
  • text-align: alinhamento de texto.
    • text-align: center;
  • font-family: fonte do texto (do mais específico para o mais genérico).
    • font-family: 'Courier New', Courier, monospace;
  • background-image: imagem de fundo. Seu valor é a url da imagem pretendida:
    • background-image: url("paper.gif");

  1. Propriedades mais Comuns

A propriedade position do CSS é uma propriedade para posicionar elementos em uma página da web. Ele oferece quatro valores principais que controlam como os elementos são posicionados em relação ao fluxo normal do documento:

  • Static: Este é o valor padrão. Elementos com esse valor são posicionados de acordo com o fluxo normal do documento. Ignora quaisquer propriedades de posicionamento (esquerda/direita/topo/fundo).
  • Relative: : Posiciona o elemento em relação à sua posição normal. Isso significa que você pode mover o elemento de sua posição normal usando as propriedades top, right, bottom e left. No entanto, o espaço ocupado pelo elemento permanece o mesmo no layout da página.
  • Absolute: Posiciona o elemento em relação ao seu elemento pai mais próximo que tem uma posição diferente de static. Se não houver nenhum elemento pai posicionado, ele será posicionado em relação ao < html>. Isso permite que você mova o elemento de forma absoluta, ignorando o fluxo do documento.
  • Fixed: posiciona o elemento em relação à janela de visualização do navegador. Isso significa que o elemento permanece fixo mesmo quando a página é rolada. É útil para cabeçalhos de página ou menus de navegação que você deseja manter sempre visíveis.
